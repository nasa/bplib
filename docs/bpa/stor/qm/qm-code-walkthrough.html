Walkthrough 1 - Forward Bundle

Client calls BPLib_STOR_QM_AllocQueueTable to init queue_tbl and cache

Client calls BPLib_STOR_QM_CreateRamStorage with cache pool from queue_tbl

Which calls:

  • BPLib_STOR_QM_Attach
  • BPLib_STOR_CACHE_Init to fully initialize cache

Client calls BPLib_STOR_QM_ClaIngress - which calls:

  • BPLib_STOR_QM_GetIntfControlblock - Duct interface
  • BPLib_STOR_QM_GenericBundleIngress
  • BPLib_STOR_QM_ReleaseIntfControlblock
  • BPLib_STOR_QM_SetMaintenanceRequest

Cache state machine drives bundle and job queues

Cache state machine calls BPLib_STOR_QM_ClaEgress - which calls:

  • BPLib_STOR_QM_SetMaintenanceRequest - last chance for egress
  • BPLib_STOR_QM_GetIntfControlblock - Duct interface
  • BPLib_STOR_QM_GenericBundleEgress
  • BPLib_STOR_QM_ReleaseIntfControlblock

Walkthrough 2 - QM Alloc and Init

Client calls BPLib_STOR_QM_AllocQueueTable to init queue_tbl and cache - which calls:

  • calloc - memory alloc currently done in QM so it isn't stubbed.
  • BPLib_STOR_CACHE_CreatePool uses MEM for calloc when calloc not in QM.
  • BPLib_STOR_CACHE_InitListHead - init duct_list (also called duct table or duct subqs)

Walkthrough 3 - Cache Finite State Machine (FSM)

States: Undefined, Idle, Queue, Delete - for now

QM calls BPLib_STOR_CACHE_SubqWorkitemInit

initial state is Idle

FSM enqueues bundle and changes state to Queue by conditions

  • BPLib_STOR_CACHE_FsmStateIdleEval sets state
    • BPLib_STOR_CACHE_FsmStateQueueEnter
      • BPLib_STOR_CACHE_RefMakeBlock
      • BPLib_STOR_CACHE_GetDuct
      • BPLib_STOR_QM_DuctTryPush
      • (conditionally) BPLib_STOR_CACHE_RecycleBlock
        Job scheduler dequeues bundle and calls action function
  • BPLib_STOR_CACHE_ActionFunc_t

FSM evaluates bundle state periodically

  • Ref counts, state conditions
  • Stay in Queue state or transition to Idle or Delete
  • Delete is the only state that transitions to Undefined
